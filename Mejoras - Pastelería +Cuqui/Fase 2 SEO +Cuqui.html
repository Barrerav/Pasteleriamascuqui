og:image · Open Graph

Preview al compartir

En Next.js 14+ con App Router, las metadata de og:image se definen en layout.tsx o en cada page.tsx. Te muestro ambas opciones.

Opción A — metadata estática en layout.tsx recomendada

La más sencilla. Define los metadatos una vez y se aplican a toda la web.

📁 src/app/layout.tsx
import { Metadata } from 'next'

export const metadata: Metadata = {
  title:       'Más Cuqui — Pastelería & Cafetería Artesanal',
  description: 'Pastelería artesanal en Ciudad Real. Tartas personalizadas, desayunos y café de especialidad. Obrador propio, elaborado cada día con amor.',

  // ── Open Graph (WhatsApp, Facebook, LinkedIn) ──
  openGraph: {
    title:       'Más Cuqui — Pastelería Artesanal · Ciudad Real',
    description: 'Tartas personalizadas, desayunos y café de especialidad. Obrador propio.',

¿Qué foto usar como og:image?

La imagen debe ser 1200×630px (ratio 1.91:1). Aparecerá como miniatura en WhatsApp, así que tiene que ser la foto más apetecible que tenga la clienta. Recomendaciones:

🎂
Tarta de cumpleaños
Alta conversión — producto estrella, colores vivos
☕
Arte latte
Muy compartible, transmite calidad artesanal
🥐
Bollería / cachitos
Diferenciador venezolano, genera curiosidad

📐 Cómo preparar la imagen

Recorta la foto a 1200×630px (puedes hacerlo en Canva, gratis). Guárdala como public/images/og-image.jpg con calidad media (70%). No hace falta que sea ultra-pesada.

🧪 Cómo verificar que funciona

Después del deploy, ve a developers.facebook.com/tools/debug y pega la URL. Te mostrará exactamente la preview que verán en WhatsApp y Facebook.

import { Metadata } from 'next'
