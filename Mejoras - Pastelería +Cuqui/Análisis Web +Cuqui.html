Mejoras detectadas

Crítico Hero — Quitar widget de Instagram, poner vídeo

El hero actual muestra una simulación de perfil de Instagram (foto, seguidores, grid de posts). Es creativo pero confuso — el visitante no sabe si está en una web o en Instagram. El texto "Donde todo se hace con amor" y los CTAs aparecen a la derecha del widget, sin la prominencia que merecen.

🔴 Ahora
Widget Instagram (mock) a la izquierda + texto a la derecha. Confuso, impacto dividido, no hay vídeo.
🟢 Propuesta
Vídeo a pantalla completa (reel de Instagram o vídeo propio) + overlay + texto centrado + CTAs. Impacto total.
→ Usar un reel corto de Instagram (tartas, lattes, cachitos) o grabar un vídeo de 15-30s en el obrador
→ <video autoplay muted loop playsinline src="/videos/hero.mp4"> — el autoplay sin sonido funciona en todos los navegadores
→ Fallback: si no hay vídeo aún, usar una foto grande de tarta/producto como fondo temporalmente
Crítico Sección "Nuestra historia" — Foto de Nintendo Switch

En el bloque con 3 imágenes junto al texto de la historia de Liser, hay una foto de una Nintendo Switch con Super Mario. Seguramente es de un post de Instagram sobre una tarta temática de Mario, pero lo que se ve es la consola, no la tarta. Rompe completamente el hilo visual de la pastelería.

→ Sustituir esa imagen por una del obrador, de Liser trabajando, o una tarta bonita
→ Idealmente las 3 fotos de esa sección deberían mostrar: obrador / producto estrella / ambiente del local
Crítico Sección "Productos" — Solo iconos, sin fotografía

Las 4 tarjetas de producto (Cafetería, Pastelería, Desayunos, Pedidos) tienen icono + texto + precio. Para una pastelería artesanal, no mostrar fotografía de producto es una oportunidad perdida enorme. Las fotos ya existen en la galería.

→ Añadir una foto de cabecera por tarjeta (ratio 4:3): latte art, tarta de cumpleaños, cachito venezolano, cupcakes
→ El icono actual puede quedar como overlay o eliminarse — la foto lo sustituye con más impacto
Media og:image — Preview al compartir por WhatsApp

Cuando la clienta o sus clientes comparten el link en WhatsApp o Instagram Stories, la preview que aparece depende del og:image. Si no está bien configurado, la preview sale genérica o sin foto — y esta web se comparte mucho.

→ En layout.tsx: añadir openGraph.images con una foto apetecible (tarta o latte art, 1200×630px)
→ También añadir twitter:card: "summary_large_image" para Twitter/X
Media SEO local — Schema.org para Google Maps y búsquedas

No hay datos estructurados JSON-LD con schema Bakery. Google Maps y la búsqueda local los usan para mostrar horarios, dirección y rating directamente en resultados de búsqueda, sin que el usuario entre a la web.

const schema = {
  "@type": "Bakery",
  "name": "Más Cuqui",
  "address": { "streetAddress": "Calle Tinte 17 Bajo", ... },
  "openingHours": ["Mo-Fr 08:30-20:30", "Sa 09:00-14:00"],
  "aggregateRating": { "ratingValue": "4.8", "reviewCount": "164" }
}
Menor Galería — Sin lightbox al hacer click en las fotos

La galería muestra el grid de fotos muy bien, pero al hacer click no pasa nada. El usuario no puede ver las fotos ampliadas. Con fotos tan buenas de producto, un lightbox simple aumentaría mucho el tiempo en la página.

→ Instalar yet-another-react-lightbox o similar — es ligero y sin configuración
Menor Contadores — Arrancan en 0 antes de scroll

Los contadores animados de la sección oscura ("4 años", "3K seguidores"…) se activan con scroll pero si el usuario llega a esa sección sin hacer scroll mínimo, los ve en 0 unos instantes. Da sensación de carga incompleta.

→ Verificar el threshold del IntersectionObserver — ponerlo a 0.2 y añadir un fallback que muestre el valor final si pasan 2s sin activarse
Para ti (dev) Sistema de gestión de fotos centralizado

La clienta querrá cambiar fotos frecuentemente. Centralizar todas las referencias a imágenes en un solo archivo de datos hace que cualquier cambio sea cuestión de editar una línea, sin buscar en componentes.

// src/data/photos.ts  ← único archivo que editas

export const heroVideo = "/videos/hero.mp4";      // ← cambiar vídeo aquí
export const heroPoster = "/images/hero-poster.jpg"; // fallback imagen

export const productPhotos = {
  cafeteria:  "/instagram/latte-art.jpg",
  pasteleria: "/instagram/tarta-cumple.jpg",
  desayunos:  "/instagram/cachitos.jpg",
  pedidos:    "/instagram/cupcakes.jpg",
};

export const historyPhotos = [
  { src: "/instagram/obrador.jpg",  alt: "Obrador artesanal" },
  { src: "/instagram/latte-2.jpg",  alt: "Arte latte" },
  { src: "/instagram/tarta-2.jpg",  alt: "Tarta de temporada" },
];

export const galleryPhotos = [
  // Añade o quita fotos aquí sin tocar el componente Galería
  { src: "/instagram/post-1.jpg", alt: "...", cat: "cafeteria" },
];
→ Renombrar las fotos descriptivamente: tarta-mario.jpg en vez de post-4.jpg — así sabes qué hay sin abrir la imagen
